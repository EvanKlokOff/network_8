записи рассматриваются по порядку

ACL, настроенный на интерфейсе, то мы можем повлиять на процесс маршрутизации. Перед поиском маршрута, будет совершен поиск разрешений в ACL

Здесь суть такая:
* что стандартный ACL располагай ближе к сети назначения, чтобы отсечь правила, как можно меньше сетей

* что расширенный ACL располагай ближе к сети источника, чтобы забанить запрещенный трафик как можно раньше и не тратить ресурсы оборудования на его маршрутизацию, доставку

### Настройка стандартного номерованного ACL
**Трафик, генерируемый роутером не обрабатывается ACL. ACL только для проходящего трафика**

**В стандартном ACL(номерованном и именованном) существует неявное последнее правило отбрасывания всего трафика deny any**

**Используй верхний регистр для именования acl**

**Название правил уникально**

* номер списка доступа определяет какой список будет назначаться стандартный или расширенный. **Для стандартного используй от 1 до 99 илл от 1300 до 1999**
* действие над правилом:
  * deny - отклонить
  * permit - разрешить
  * remark - комментарий, для того, чтобы указать причину создания правила
* указываем ip источника + обратную маску
* Назначение на физический интерфейс:
  * in/out - направление трафика(входящий/исходящий)
* Назначение на виртуальный интерфейс(для контроля удаленным доступом)

### Редактирование ACL

![alt text](image-38.png)

Если acl уже создан и настроен на интерфейс, то просто дописать нужное правило не удасться, оно просто добавится в конец записей acl

Если нужно отредачить запись acl, то придётся удалить acl и пересоздать его заново с нужным упорядоченным набором правил

предварительно нужно удалить acl с интерфейсов, иначе на интерфейсе будет отбрасываться весь трафик

Работает для именнованных, либо для номерованных, если считать номер именем acl(**в примере вместо имени пиши номер**)

Каждому правилу в acl присваивается номер, по которому можно таргетно редактировать правило(удалить и создать его заново с указанием нужного порядкового номера правила)
