# PPPoE (PPP over Ethernet)

![alt text](image-9.png)

PPP используется только для последовательных каналов связи, поэтому ,когда ethernet стал набирать популярность, изобрели PPPoE

Для этого ppp пакет помещается внутрь ethernet пакета, для прохождения по ethernet сети. Далее при прохождении граничного роутера, ethernet заголовок извлекается и работает уже обычный ppp

* Большой плюс, что не нужно настраивать новую инфраструктуру для нового протокола, а также делать большие изменения в старой

## Настройка PPPoE(для клиента)

Пример настройки клиентского роутера

Интерфейс dialer - связь между ethernet и последовательным каналом связи

`ip address negotiated` - адрес требуется согласовать с противоположной стороной, т.е он будет получен автоматически

Данные для авторизации необходимо продублировать на противоположной стороне(провайдер)

Нужно установить mtu меньше чем 1500(1492), т.к требуется зарезервировать место для заголовка ppp, которого нет в пользовательских данных

Нужно создать пул, который связан с физ-им интерфейсом, для привязки виртуального интерфейса к физическому

Далее заходим на физический интерфейс ethernet и включить pppoe и привязать этот физический интерфейс к виртуальному. **Включи его обязательно**. IP адресс вводить не нужно, т.к ранее ввели `ip address negotiated`

![alt text](image-10.png)

### Приколы с MTU. Откуда 1492 

Стандартный ethernet имеет 1500 байт в кадре. Но нужно учесть размеры заголовков(tcp + ip). Поэтому на данные остаётся 1460 байт

Но помимо данных ethernet есть ещё ppp заголовок(8 байт), поэтому нужно выделить место для его передачи

![alt text](image-11.png)

получается что максимальный размер данных в кадре 1452.

Чтобы согласовать размер кадра используй команду. Размер сегмента 1452

## BGP(Протокол Граничного Шлюза)

Основной протокол внешней маршрутизации для построения маршрутов между автономными системами
* Тип протокола - Path vector
![alt text](image-20.png)

### Типы BGP

ibgp - это не протокол внутрненней маршрутизации, он просто передаёт трафик, который был получен внешним протоколом ebgp

ibgp рассматриваться не будет в этом семе

### Варианты работы BGP

Самый подходящий тип подключения для bgp это multihomed
![alt text](image-24.png)

Варианты:
* Предоставление маршрута по умолчанию
  * У компании есть доступ к инету
  * У компании A Нет понимания, через какой провайдер нужно отправить трафик, чтобы маршрут был более оптимальный
* Маршрут по умолчанию + свои сети
  * У компании А появляется четкое видение сетей АС, поэтому можно оптимизировать маршруты до них
  * Мы не знаем какие сети расположены в остальной части Internet, поэтому для них маршрут оптимизировать не удасться
* Провайдер объявляет все известные ему маршруты
  * Можно строить ещё более оптимальные маршруты, чем раньше
  * Появляется проблема с хранением большого кол-ва инфы о сетях, придётся более дорогое оборудование покупать
  * Как вариант, можно показывать не все маршруты, а только те, которые нужны компании A

### Настройка eBGP(вариант когда компания связана с одним ISP) 

Роутер может находится только в одной АС

Вводим на граничном роутере, в качестве номера ас вводим номер локальной АС, в которой находится этот роутер
![alt text](image-26.png)

Заходим в настройку маршрутизации и объявляем однорангового равноправного соседа, с которым происходит обмен трафиком
* ip адрес это глобальный IP
* as-number - номер АС, с которой хотим производить обмен
![alt text](image-27.png)

Прописываем сети, которые будем распространять
* Если сеть отличается от классовой сети, то вводим маску
* Сеть не обязательно должна принадлежать интерфейсу роутера, это может быть каким-то способом выученная сеть, находящаяся в таблице маршрутизации
